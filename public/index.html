HotSCAD

OpenSCAD for agentic CAD design: every save renders. Instant feedback whether Claude is at the keyboard or you are.

Open a .scad file on your computer, edit it in your favorite editor or hand it to Claude, Codex or Gemini and HotSCAD re-renders the preview the moment the file is saved. No copy-paste. No reload. Nothing to install.

Your files never leave your computer.

HotSCAD reads the file you pick using the browser's File System Access API. Nothing is uploaded anywhere. Every render, every preview, every customizer value runs locally in your browser. The site itself is static, there's no backend to send anything to.

What is HotSCAD?

This is a fork based on openscad-playground by Olivier Chafik, which solved the speed problem first by getting OpenSCAD to run in the browser. I wanted to further improve the user experience to iterate faster and more efficiently. Everything below is what HotSCAD adds on top.

Getting started

Just open and go.

Open a file with hot reload in the browser

Click Open Local… in the editor toolbar and pick a .scad on your computer. HotSCAD watches the file and re-renders every time you save it, so it's easy to let Claude / Codex / Gemini do the editing.

Watching, in sync
Manual reload mode (Firefox / Safari)
Editor differs from disk - click reload to pull latest

Dark mode (finally)

The whole app now renders dark by default. Switch to light any time from Settings; HotSCAD remembers your choice and follows your OS preference if you leave it on Auto.

The View tab

Navigate easier and faster.

Camera steady on re-render

Tweaking a parameter doesn't snap the view back to default. The camera holds its position through every render, so iterating on small details actually works.

Empty-render warning

When the source compiles but produces no geometry, Current top level object is empty is shown in the Console (which is hidden by default) but the viewer keeps the previous model on screen and shows a warning at the top explaining why nothing changed.

ViewCube

Click a face (Front, Top, a corner) to snap to that view. Drag the cube to orbit. Same as Fusion.

Two buttons. Home snaps back to a 3/4 view. Fit frames the current model exactly to the panel.

LMB pan, RMB rotate (and more in Settings)

Fusion-style mouse mapping by default. Pan with the primary mouse button, orbit with the secondary. Scroll to zoom.

In Settings you can swap the mapping and tweak other things like the debounce that sets how quickly the model re-renders while you're changing parameters.

WASD navigation

Game-style mouse/keyboard navigation. Hold RMB to rotate and aim where you want to go, and press W to move forward.

Pressing Q and E moves down and up.

Show dimensions

A toggle prints the bounding-box dimensions next to each component, in millimeters. Useful for checking print volume without opening a slicer.

The Customize tab

Parameters and per-layer color

Auto font dropdown

Any string parameter ending in font turns into a dropdown of bundled fonts — Inter Black / SemiBold, Liberation Sans / Serif / Mono, Noto Sans. You can still type a custom font name; the dropdown is editable.

Quick parameter settings

The parameter panel has been redesigned to be easier to use with new smart features.

  • Filter parameters - Search for a parameter for easy access

  • Revert all values - When you've slid every slider down a rabbit hole, one click puts every customizer override back to the defaults declared in the source file.

  • Show Customized - Hide all parameters you haven't changed, easier to focus

Click to nudge, Shift-click for fine

Every number input has spinner arrows. Click to step the value by 1; Shift-click (or hold Shift and use the arrow keys) to step by 0.1. Faster than dragging a slider when you know roughly where you're going.

Layer-colors panel

Preview how the model looks with filament colors swapped at specific Z heights. Useful for planning a two- or three-color print without going to the slicer.

A note before you start agent-coding your CAD

Results vary, a lot. Different LLMs handle OpenSCAD with different levels of competence, and even the good ones occasionally hand you back something that looks nothing like what you asked for. Sometimes a part shows up in seconds; sometimes the agent will burn fifteen minutes of back-and-forth chasing a single chamfer.

If you already know CAD and you're optimizing for speed, this workflow can feel frustrating. If you don't know CAD — or just don't want to learn it — but you still want to design and print real things, this is the easiest way in. Describe what you want in plain English, iterate on the result, print it. That's the loop HotSCAD is built around.

Run it locally

For hacking, forking, or running offline

You don't need to install anything to use HotSCAD — just open HotSCAD in your browser. But if you want to hack on it or run it on your own machine:

  • Node.js ≥ 18.12
  • npm
  • git
  • zip
# Clone the repo
git clone https://github.com/nicmar/hotscad.git
cd hotscad

# Download OpenSCAD WASM + bundled libraries (one-time, takes a minute)
npm run build:libs

# Install JS deps
npm install

# Start the dev server at http://localhost:4000/
npm run start

# Production build into ./dist
npm run build:all

That's all there is.
